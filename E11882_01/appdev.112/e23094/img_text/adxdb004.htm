This figure shows a tree structure with parent nodes and leaf nodes (nodes without children). The root is labeled "/ - Root Node", and has two children: home and sys.

  • Node home has two children: QUINE and XDB. Node QUINE has five children: po, general, graphics, schemas and acls (QUINE/schemas and QUINE/acls are leaves). Node po has three leaf children, collectively labeled "XML files stored in your XMLType tables/views": po_101.xml, po_Jan02.xml, and po_Jan03.xml. Node general has five leaf children, collectively labeled "Binary files stored as BLOBs": readme.txt, whatsnew.fm, into.doc, maincode.jav, and chapter1.xml. Node graphics has three leaf children, collectively labeled "Graphics or binary files stored in BLOBs": architecture.jpg, logo.gif, and banner.png. Node XDB has two leaf children: acls and schemas.

  • Everything under node sys is collectively labeled "Oracle XML DB System Folders (Do not store your data in sys)". Node sys has three children: acls, log, and schemas. Node sys/acls has two leaf children, collectively labeled "ACL files are stored in xdb.xdb$ACL table": all_owner_acl.xml and all_all_acl.xml. Node log has one leaf child: 11_28_01.txt. Node sys/schemas has one child: PUBLIC. Node PUBLIC has one child: xmlns.oracle.com. Node xmlns.oracle.com has one child: xdb. Node xdb has one leaf child: XDBSchema.xsd. Nodes schemas, PUBLIC, xmlns.oracle.com, and xdb are collectively labeled "Oracle XML DB folders (containers)". Leaf node XDBSchema.xsd is labeled "XML files typically stored object-relationally can also store in LOBs".

All nodes in the figure with extensions, such as .txt, .jav, and .xml, represent files or documents. The other nodes (including xmlns.oracle.com) represent directories (containers).
